# 档案页面生成规范 (Step 2)

基于检索到的信息，生成一个完整的 React 组件页面。

## 页面结构

```
├── Navbar (透明→实心, 品牌标识 + 返回按钮)
├── Hero Section
│   ├── 人物肖像区 (大图/水墨风背景)
│   ├── 姓名 + 字/号
│   ├── 朝代 + 身份标签
│   └── 一句话概括
├── 生平时间线 (Timeline)
│   ├── 以时间线组件展示人生重要节点
│   ├── 每个节点: 年份 + 事件 + 简要说明
│   └── 交互: hover 展开详情
├── 成就与贡献
│   ├── 网格布局, 3-4 张成就卡片
│   └── 每张: 图标 + 标题 + 描述
├── 代表作品/名句
│   ├── 卡片式展示, 带引用样式
│   └── 名句卡片特殊处理: 大号书法风格字体
├── 历史评价
│   ├── 后世评价引用
│   └── 评价人 + 来源标注
├── Stats Bar
│   ├── 3-4 个数据点 (如存世诗篇/生卒年份/作品数量等)
│   └── 数字高亮展示
└── Footer
    └── 品牌标识 + 版权信息
```

## 页面设计规范

### 配色方案

| 角色 | 色值 | 用途 |
|------|------|------|
| 主背景 | `#0F1419` | 深青黑 |
| 主文字 | `#F0EDE8` | 暖象牙白 |
| 点缀/按钮 | `#5B8A72` | 青铜绿 |
| 链接/高亮 | `#6B9BD1` | 天青 |
| 标题/印章 | `#C9A96E` | 琥珀金 |
| 标签/强调 | `#B85C50` | 朱砂 |

### 主题色（按朝代自动匹配）

| 朝代 | 主题色 |
|------|--------|
| 先秦秦汉 | 青铜绿 `#5B6B4D` |
| 魏晋隋唐 | 琥珀金 `#C9A04E` |
| 宋代 | 天青 `#5BA3C4` |
| 元明清 | 朱砂 `#A85A4D` |

### 玻璃态 UI

```css
.artifact-glass {
  background: rgba(255, 255, 255, 0.05);
  backdrop-filter: blur(20px);
  border: 1px solid rgba(255, 255, 255, 0.1);
}

.artifact-glass-strong {
  background: rgba(255, 255, 255, 0.08);
  backdrop-filter: blur(30px);
  border: 1px solid rgba(255, 255, 255, 0.15);
}
```

### 字体

- **标题**: Noto Serif SC (衬线，庄重感)
- **正文**: Noto Sans SC (无衬线，可读性)

### 动画

- **库**: Framer Motion
- **缓动函数**: `[0.22, 1, 0.36, 1]` (克制优雅)
- **原则**: 动画保持克制，支持 `prefers-reduced-motion`

### 图标

- **库**: lucide-react

## 技术栈

```
React 19 + TypeScript + Vite 8 + Tailwind CSS 3.4.1
+ framer-motion (动画)
+ lucide-react (图标)
+ EdgeOne Pages (部署)
+ Edge Functions (API)
```

## 代码生成要点

1. 使用 Tailwind CSS 类，不写独立 CSS 文件
2. 玻璃态效果使用上述 `artifact-glass` 类
3. 每次生成只输出 `src/App.tsx` 单文件组件
4. 数据部分硬编码在组件内（暂不拆分 JSON 文件）
5. 人物图片使用 Unsplash 或占位符，避免版权问题
6. 支持 dark mode（默认即为暗色主题）
7. 全响应式布局（移动端适配）
